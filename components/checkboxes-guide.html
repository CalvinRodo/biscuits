Checkboxes: Design guidelines

Checkboxes let people select one or more options from a list of items.

Design specifications

  • 1. Question or heading

    The legend preceeding large checkboxes inputs should use h5 styles. These are applied by the chkbxrdio-grp class applied to the parent fieldset.

  • 2. Checked checkbox input

    A checked checkbox input should use a checkmark icon with 5px width, 3px for small checkboxes.

    The green checkbox variant to indicate successful completion of an item in a checklist, uses a #318000 green background colour.

  • 3. Unchecked checkbox input

    An unchecked checkbox appears with 36px by 36px dimensions, 24px by 24px for small checkboxes, and a black, 2px outline.

  • 4. Checkbox label

    Checkbox label should appear with 20px padding after the checkbox input. It uses 20px font size, 17px for small checkboxes, and maintains it's right side indentation when text wraps.

  • 5. Continue button

    A continue button should always come after a checkbox used for an interactive question and should generally use the primary button styling with a btn-primary class.

Usage

Patterns for contextual alerts

  • Notifications with alerts, overlays and dialogs
  • Banners

Don't

Don't use a legend before a checklist (not answering a question or in a form). If needed, a checklist can be preceeded by a heading with a h5 class.

Do

A checklist does not require a form action. The purpose of a checklist can be to support people tracking items they have completed. However, make sure to add an paragraph using wb-inv above the checklist to explain to screen readers that toggling the checklists is simply to track ticked off items.

Do

Place an alert above the page heading if it applies to more than one page.

Do

Place an alert underneath a page heading if it applies to the whole page.

Do

Place an alert within a section of a page where it is relevant.

Do

If an alert advises of an issue or risk, it must provide a next step or suggested course of action to pursue.

Don't

Don't use an alert for content that will only be relevant to a small percentage of visitors to the page.

Do

Detect a user's situation to make an alert appear only to those it affects.

Don't

Don't shout (use caps) or use italics to call attention.

Don't

Alert messages should be polite and use plain language (~6-8 grade reading level).

Don't

Don't use an alert for form validation. Use for validation instead.

Consider labels or pills for styling of text with emphasis colours.

Do not use contextual alerts:

  • to provide information about a normal step in a process or task
  • to simply add emphasis to content
  • in cases where a warning is neither high risk nor frequently experienced
  • to create labels or pills. For these components, see:
    • label pattern
    • pill pattern

How to use

Alerts should be written for a grade 6-8 reading level.

Place the alert in the appropriate context:

Applies to the entire site
Place it at the top of the page above the main heading
Applies to the page
Place it underneath the page heading
Applies to a subsection of the page
Place it within that subsection, generally underneath the subsection heading or between 2 paragraphs

You can either use the beta or stable of version of this component. The beta version will eventually replace the stable one.

Use the beta contextual alerts component

Note: You will need to use custom CSS to implement these beta patterns.

Danger alert

When to use: Use the danger alert to draw attention to a situation that could place the user in danger, such as a do not travel warning. You can also use it to alert the user of a technical issue that could cause them to be unable to complete their task, such as an invalid submission of data in a form.

If pregnant or considering pregnancy - both partners

Avoid travel to Costa Rica if you are pregnant or planning to conceive a child in the next 3 months.

Code
HTML 
<section class="alert alert-danger">
 <h3>If pregnant or considering pregnancy - both partners</h3>
 <p>Avoid travel to Costa Rica if you are pregnant or planning to conceive a child in the next 3 months.</p>
</section>
CSS 
.alert:before {
 height: 34px;
 content: "";
 background-color: #fff;
 width: 6px;
 display: inline;
 margin-top: -1px;
 margin-left: -26px;
 position: absolute;
}
.alert-success, .alert-info, .alert-warning, .alert-danger {
 background: #fff !important;
}
.alert {
 border-width: 1px 1px 1px 4px !important;
 margin-left: 12px;
 padding-left: 20px;
 border-width: 0px 0px 0px 6px!important
}
.alert-success > :first-child::before, .alert-info > :first-child::before, .alert-warning > :first-child::before, .alert-danger > :first-child::before {
 font-size: 26px;
 margin-left: -1.48em;
 padding: 2px;
 line-height: 1.1em;
}
.alert > :first-child {
 margin-left: 0em;
}
.alert >:first-child::before {
 margin-right: 0.4em
}

Warning alert

When to use: Use the warning alert to draw attention to a possible consequence of an action or inaction, such as legal penalty that could apply.

Changes to impaired driving and cannabis-related crime laws

New penalties for impaired driving and cannabis-related crimes may affect your immigration status in Canada. If you commit one of these crimes, you may not be able to come to or stay in Canada.

Code
HTML 
<section class="alert alert-warning">
 <h3>Changes to impaired driving and cannabis-related crime laws</h3>
 <p><a href="https://www.canada.ca/en/immigration-refugees-citizenship/news/notices/impaired-driving-cannabis-penalties-affect-immigration-status.html">New penalties for impaired driving and cannabis-related crimes may affect your immigration status in Canada</a>. If you commit one of these crimes, you may not be able to come to or stay in Canada.</p>
</section>

Success alert

When to use: Use success alert to draw attention to a successful action, such as to confirm a submission or the completion of a task.

The turnaround time to process your request may take up to four (4) weeks. We will contact you if we need more information.
