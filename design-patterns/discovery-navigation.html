Discovery navigation: Design pattern

Options to let people find the task they need to accomplish from a navigation page.

Available components

1. Link and description

Use the link and description pattern to show unordered links of topics, services or tasks.

2. Sequential link and descriptions

Use a single column list of numbered links and descriptions when there are steps people are familiar with that can help identify which navigation option to follow.

3. Link and description with dropdown menu

Consider adding a dropdown menu to the link and description pattern to give a shortcut to task pages, if there is a table or long list of links on the subsequent page.

4. Radio buttons

Use an interactive question with radio buttons to navigate to closely related options, to help contrast differences and avoid duplicate keywords in multiple labels.

5. Bulleted link(s)

Use a list item to make links stand out

6. Buttons

Usage

Allow alternative input or visualisation for sliders

Do

Consider using a slider to answer with a number, but allow alternative direct input.

Single item checkboxes

Don't

Don't use a single item checklist when the answer is frequently different.

Do

Instead of a single item checklist, use yes/no radio buttons to ensure a conscious selection is made.

Do

Consider single item checklists for a yes/no answer that is usually answered the same way, or to nudge with a default.

Error validation

Do

Customise validation messages to make fixing errors easier.

Do

When validating numbers, ensure that all common number formats are supported, such as using a space or comma between groups of 3 digits.

Use of overlays for clarification

Only ask a question or make a different choice when it makes a difference

Avoid inline styles

Don't

Avoid inline checkboxes or radio buttons in interactive questions. Even for yes/no answers. Inline radio buttons should only be used for toggle states.
